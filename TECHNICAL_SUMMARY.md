# Cattle GHG Uncertainty Calculator — Technical Summary

**Version:** 2.1 | **Date:** April 2026
**Developed by:** CIAT / CGIAR Alliance | **Funded by:** Global Methane Hub (Grant R-2026-01051)

---

## 0. What the App Does in One Paragraph

The app takes a national cattle GHG inventory structured as a set of animal sub-categories (e.g. dairy cows, heifers, growing males) and, for each sub-category, a list of input parameters with their uncertainty. It runs the **IPCC 2006 Tier 2 equation chain** (Volume 4, Chapter 10 and 11) 10 000 times, each time drawing a slightly different set of inputs, to produce a distribution of possible emission values rather than a single number. Activity data parameters are sampled using a **Gaussian copula** that preserves their pairwise correlations (estimated automatically from historical time series supplied in the template); emission factors can be sampled independently (IPCC default) or with a **uniform equicorrelation** to represent systematic methodological bias. From the resulting distribution the app reports the mean emission, the 95 % confidence interval, the coefficient of variation (CV %), a decomposition of how much uncertainty comes from activity data vs. emission factors, and a sensitivity ranking showing which parameters drive uncertainty most. An optional **comparison run** re-runs the simulation without any correlations so users can quantify the effect of correlation assumptions. Results map directly to IPCC Volume 1 Chapter 3 uncertainty reporting requirements.

## 2. Input Data — What the User Must Provide

The user uploads one Excel workbook with up to 6 sheets. **Only the `Parameters` sheet is mandatory.** The parser is in `parse_uploaded_template()` (`R/utils_template.R`). The template is generated by `generate_template()`, which calls `generate_template_openxlsx()`.

### 2.0 Template Design and User Interface

The template is designed to be filled correctly at first glance: every column that requires a controlled value has a dropdown menu, every numeric column that can be auto-computed has an Excel formula, and colour-coding distinguishes what the user must type from what is pre-filled or calculated.

#### Central Parameter Catalogue — `PARAM_CATALOGUE`

Declared at the top of `R/utils_template.R`, `PARAM_CATALOGUE` is a **24-row** data frame that serves as the single source of truth for the Parameters sheet (pre-populated rows) and the Vocab sheet (reference tables). Columns:

| Column | Description |
|---|---|
| `parameter` | The 24 recognised parameter names |
| `definition` | Plain-language description of the variable |
| `unit` | Unit string (e.g. `kg`, `MJ/day/kg^0.75`, `fraction`) |
| `ipcc_default` | IPCC default numeric value; `NA` for `cattle_pop` (no universal default) |
| `suggested_uncertainty_pct` | Recommended +/- % for symmetric distributions; `NA` for asymmetric parameters (use bounds instead) |
| `suggested_distribution` | Recommended distribution shape (e.g. `pert`, `normal`, `lognormal`) |
| `suggested_lower_bound` | Absolute lower bound for asymmetric parameters per Penman et al. (2000) / Monni et al. (2007); `NA` for symmetric parameters |
| `suggested_upper_bound` | Absolute upper bound for asymmetric parameters (same source); `NA` for symmetric parameters |
| `param_type` | `activity_data` or `emission_factor` |
| `param_tier` | `core` (user must provide) or `technical` (IPCC coefficient, pre-filled with default) |
| `user_reducible` | `TRUE` if improving local data reduces uncertainty; `FALSE` for IPCC coefficients requiring dedicated research |
| `ipcc_ref` | IPCC table or equation reference (e.g. `Table 10.4`, `Eq 10.24`) |

### 2.3 The 24 Recognised Parameter Names

These are the exact strings the model looks for. Any row whose `parameter` column does not match one of these is ignored.

#### Activity data (AD) — 15 parameters; sampled with copula if correlation matrix exists

| `parameter` | Definition | Unit | Default source |
|---|---|---|---|
| `cattle_pop` | Number of animals in sub-category | head | Country census / FAOSTAT — no IPCC default |
| `live_weight` | Average live body weight | kg | IPCC Table 10A.2; Africa defaults in `LW_BY_SUBCAT` |
| `mature_weight` | Mature adult body weight | kg | IPCC Table 10A.2; `MW_BY_SUBCAT` |
| `weight_gain` | Average daily weight gain | kg/day | Table 10A.1; `WG_BY_SUBCAT`; 0 for adult non-growing |
| `milk_yield` | Daily milk yield per lactating cow | kg/head/day | Country dairy stats; IPCC regional range |
| `milk_fat` | Milk fat content | % | IPCC default 4 %; country dairy stats preferred |
| `pct_lactating` | Fraction of cows currently lactating | fraction 0–1 | App default 0.60; herd structure surveys |
| `DE_pct` | Digestible energy as % of gross energy | % | IPCC Table 10.2 (45–75 %); feed lab analyses preferred |
| `Cfi` | Maintenance energy coefficient | MJ/day/kg^0.75 | **IPCC Table 10.4** — `CFI_BY_SUBCAT`: lactating cow 0.386, dry cow/heifer 0.322, bull 0.370, oxen 0.322 |
| `Ca` | Activity coefficient (energy cost of locomotion) | dimensionless | **IPCC Table 10.5** — auto-filled from `feeding_situation`: stall 0.00, pasture flat 0.17, hilly 0.36 |
| `C_growth` | Growth coefficient for NEg equation | dimensionless | **IPCC Eq 10.6** — auto-filled from `sex`: female 0.8, castrate/oxen 1.0, bull 1.2 |
| `Cp` | Pregnancy coefficient | dimensionless | **IPCC Table 10.7** = 0.10 for pregnant animals |
| `hours` | Daily working hours (Eq. 10.11) | hours/day | 0 when animals do no work; farm surveys when applicable |
| `CP_pct` | Crude protein content of diet | % | Feed lab analyses; IPCC Worksheet 10.3 approx 8–18 % |
| `protein_milk` | Protein content of milk | % | Feeds the milk-N term in IPCC Vol.4 Ch.10 Eq 10.33 (N retention for cattle, where the 6.38 milk-protein-to-N conversion is defined); default 3.3 % |

#### Emission factors (EF) — 9 parameters; sampled independently or with uniform equicorrelation

| `parameter` | Definition | Unit | Default source |
|---|---|---|---|
| `Ym_pct` | % of GE converted to enteric CH4 | % | **IPCC Table 10.12**: dairy high-DE 6.0 %, dairy low-DE 6.5 %, feedlot 3.0 %, other cattle 6.5 % |
| `Bo` | Maximum CH4 producing capacity of manure | m3 CH4/kg VS | **IPCC Table 10.16**: cattle Africa 0.10, W. Europe dairy 0.24, buffalo 0.10 |
| `ash` | Ash content of manure | fraction | IPCC Eq 10.24 footnote = 0.08 |
| `UE` | Urinary energy as fraction of GE | fraction | IPCC Eq 10.24 footnote = 0.04 |
| `EF3_PRP` | N2O EF for dung/urine deposited on pasture | kg N2O-N / kg N | **IPCC Vol.4 Ch.11 Table 11.1**: 2006 = 0.02; 2019R EF3_PRP,CPP for cattle/poultry/pigs: aggregated = 0.004 (wet = 0.006, dry = 0.002); asymmetric bounds approx: [0.000, 0.014] |
| `Frac_GASM` | Fraction of manure N volatilised as NH3/NOx | fraction | **IPCC Vol.4 Ch.10 Table 10.22** varies by MMS (0.07–0.48); aggregated FracGASM in Ch.11 Table 11.3: 2019R = 0.21, 2006 = 0.20 |
| `EF4` | N2O EF for atmospheric N deposition | kg N2O-N / kg N | **IPCC Vol.4 Ch.11 Table 11.3**: 2006 = 0.010; 2019R aggregated = 0.010 (wet = 0.014, dry = 0.005); bounds approx [0.002, 0.018] |
| `EF5` | N2O EF for N leaching/runoff | kg N2O-N / kg N | **IPCC Vol.4 Ch.11 Table 11.3**: 2006 = 0.0075; 2019R = 0.011; bounds approx [0.000, 0.020] |
| `Frac_LEACH` | Fraction of managed N lost through leaching | fraction | **IPCC Vol.4 Ch.11 Table 11.3** FracLEACH-(H) (wet climate only): 2006 = 0.30; 2019R = 0.24; dry climate = 0. App default for MS-side `Frac_LEACH_H` = 0.02 (from Vol.4 Ch.10 Table 10.23) |

**Asymmetric parameters:** EF3\_PRP, EF4, EF5, and Frac\_LEACH have strongly right-skewed uncertainty per Penman et al. (2000) and Monni et al. (2007). Their `suggested_lower_bound` and `suggested_upper_bound` in `PARAM_CATALOGUE` override the symmetric +/- % formula. The QA/QC module (§7.5) warns if symmetric bounds are detected for these parameters.

### 2.5 Sheet: `Parameter_TimeSeries`

**Replaces the former `Population_TimeSeries` sheet** (v2.0 and earlier had rows = sub-categories, columns = years, which was stored but never used). The new sheet feeds directly into the activity data correlation pipeline.

**Format:** one row per year, one column per parameter. Columns match AD parameter names from §2.3. The sheet includes:

- Row 1: instruction banner (how the sheet is used)
- Row 2: parameter names as column headers
- Row 3: plain-language descriptions (informational)
- Row 4: units (informational)
- Rows 5–14: Country X example data (5 illustrative years), if `include_example = TRUE`
- Rows 15–34: 20 blank rows for user data

**Automatic correlation computation on upload:**

When a template is uploaded in Tab 1, `parse_uploaded_template()` reads the `Parameter_TimeSeries` sheet (with fallback to `Population_TimeSeries` for backwards compatibility). If at least 2 numeric columns and 5 complete rows are present, it calls `compute_correlation_from_timeseries()`:

```r
# 1. Select only numeric columns (excludes the year column)
cor_matrix <- cor(pop_numeric, use = "complete.obs")   # Pearson correlation
corr_matrix <- nearPD(cor_matrix, corr = TRUE)$mat     # nearest positive semi-definite
```

The resulting named matrix is stored in `parsed$corr_matrix` and wired to `rv$corr_matrix` in the server. The simulation log reports: "Correlation matrix loaded from Parameter\_TimeSeries (N parameters)".

**Partial time series:** columns can be omitted for parameters where no historical data exists. `expand_corr_matrix()` fills the missing rows/columns with identity rows (zero correlation with everything else), so partial historical data is still useful.

**You do not need to fill in every column.** The app handles missing columns gracefully — absent parameters are treated as uncorrelated.

## 3. Equation Chain — Calculated Step by Step

These are run inside `ghg_emissions()` (`R/calc_ghg_master.R`), called N times by `ghg_emissions_vec()`. Units and IPCC references are given for every intermediate.

### 3.1 Net Energy for Maintenance — Eq 10.3
**R:** `calc_nem(live_weight, Cfi)` in `R/calc_energy.R`

```
NEm = Cfi x LW^0.75          [MJ/head/day]
```

`LW^0.75` is metabolic body weight. `Cfi` scales it by the animal's maintenance requirements. It is the anchor of the whole energy chain — every other NE component is expressed relative to NEm or adds to GE through the same pipeline.

### 3.2 Net Energy for Activity — Eq 10.4
**R:** `calc_nea(nem, Ca)`

```
NEa = Ca x NEm                [MJ/head/day]
```

Ca = 0 for stall-fed animals (no locomotion cost), 0.17 for flat pasture, 0.36 for hilly terrain. These are **fixed IPCC Table 10.5 values** with no country-specific alternative — only the feeding situation classification itself varies.

### 3.3 Net Energy for Growth — Eq 10.6
**R:** `calc_neg(live_weight, weight_gain, C_growth, mature_weight)`

```
NEg = 22.02 x (LW / (C x MW))^0.75 x WG^1.097     [MJ/head/day]
```

Returns **exactly 0** if WG <= 0 (non-growing adults) or MW <= 0. The ratio `LW/(C x MW)` is the animal's current size as a fraction of its mature size. C = 0.8 for females, 1.0 for castrates/oxen, 1.2 for bulls.

### 3.4 Net Energy for Lactation — Eq 10.8
**R:** `calc_nel(milk_yield, milk_fat, pct_lactating)`

```
NEl = milk_yield x (1.47 + 0.40 x milk_fat) x pct_lactating    [MJ/head/day]
```

`1.47 + 0.40 x fat` converts kg milk to MJ using fat content (IPCC Eq 10.8). `pct_lactating` scales down to a whole-herd average. Returns 0 for non-dairy sub-categories.

### 3.5 Net Energy for Work — Eq 10.11
**R:** `calc_new(nem, hours)`

```
NEw = 0.10 x NEm x hours     [MJ/head/day]
```

Used only when animals perform work; zero for typical dairy / beef inventories.

### 3.6 Net Energy for Pregnancy — Eq 10.13
**R:** `calc_nep(nem, Cp)`

```
NEp = Cp x NEm               [MJ/head/day]
```

Cp = 0.10 from IPCC Table 10.7, applied pro-rata to the fraction of animals pregnant.

### 3.7 Efficiency Ratios REM and REG — Eq 10.14 and 10.15
**R:** `calc_rem(DE_pct)` and `calc_reg(DE_pct)`

```
REM = 1.123 - 4.092e-3 x DE% + 1.126e-5 x DE%^2 - 25.4/DE%
REG = 1.164 - 5.160e-3 x DE% + 1.308e-5 x DE%^2 - 37.4/DE%
```

These are polynomial fits to empirical metabolic data. They convert net energy requirements back to the digestible energy scale so everything can be expressed as a fraction of GE. **DE% is one of the most influential inputs** — changing it moves both REM and REG and therefore GE.

### 3.8 Gross Energy Intake — Eq 10.16
**R:** `calc_ge(nem, nea, nel, nep, new_energy, neg, rem, reg, DE_pct)`

```
GE = [(NEm + NEa + NEl + NEw + NEp) / REM  +  NEg / REG]  /  (DE% / 100)
                                                            [MJ/head/day]
```

**GE is the central intermediate variable.** Every downstream calculation (enteric CH4, VS, N excretion) depends directly on GE.

---

### 3.9 Enteric Fermentation CH4 — Eq 10.21
**R:** `calc_enteric_ch4(ge, Ym_pct)` in `R/calc_enteric.R`

```
EF_enteric = (GE x Ym/100 x 365) / 55.65      [kg CH4/head/year]
```

- `Ym` = % of GE that is converted to CH4 in the rumen. Default 6.5 % for most cattle (IPCC Table 10.12).
- 55.65 MJ/kg = energy content of methane.
- x 365 converts daily GE to annual.

```
enteric_ch4_total = EF_enteric x cattle_pop / 1000    [tonnes CH4/year]
```

---

### 3.10 Volatile Solids — Eq 10.24
**R:** `calc_volatile_solids(ge, DE_pct, UE, ash)` in `R/calc_manure_ch4.R`

```
VS = [GE x (1 - DE%/100) + UE x GE] x [(1 - ash) / 18.45]   [kg DM/head/day]
```

VS is the quantity of organic matter in manure available for microbial decomposition.

### 3.11 Manure CH4 — Eq 10.23
**R:** `calc_manure_ch4(VS, Bo, mms_fractions, mcf_values)` in `R/calc_manure_ch4.R`

```
EF_manure = VS x 365 x Bo x 0.67 x Sum_i (MCF_i/100 x frac_i)   [kg CH4/head/year]
```

- Summed over all MMS types i.
- `Bo` = maximum CH4 producing capacity (m3 CH4/kg VS).
- `0.67` converts m3 CH4 to kg CH4 (density of methane at standard conditions).
- `MCF_i` = % of Bo actually converted to CH4 for MMS type i (IPCC Table 10.17; see §4).
- `frac_i` = fraction of manure going to MMS i (must sum to 1 across all i).

```
manure_ch4_total = EF_manure x cattle_pop / 1000    [tonnes CH4/year]
```

### 3.12 Nitrogen Excretion — Eq 10.31–10.34
**R:** `calc_n_excretion(ge, DE_pct, CP_pct, milk_yield, pct_lactating, weight_gain)` in `R/calc_manure_n2o.R`

**Step 2 — Nitrogen intake:**
```

```


**Step 4 — Annual excretion:**
```
Nex = max(0,  (N_intake - N_retained) x 365)     [kg N/head/year]
```

## 5. Uncertainty Representation — Distribution Types

**R:** `sample_distribution()` and `transform_marginal()` in `R/utils_distributions.R`

For symmetric distributions the **standard deviation is derived from the bounds** as:
```
SD = (upper - lower) / (2 x 1.96)
```
This treats [lower, upper] as a 95 % confidence interval.

| Distribution | How parameterised | When recommended |
|---|---|---|
| **normal** | `rnorm(n, mean, SD)` | Symmetric uncertainty; large populations, body weights, milk yield |
| **posnorm** | Normal, then `pmax(0)` | Non-negative symmetric parameters where negative values are implausible |
| **lognormal** | `mu_log = log(mean)`; `sd_log = [log(upper)-log(lower)]/(2x1.96)`; `rlnorm(n, mu_log, sd_log)` | Strictly positive, right-skewed; EF4, EF5 |
| **beta** | `mu = (mean-lower)/(upper-lower)`; alpha/beta computed from mu and var; scaled back to [lower, upper] | Parameters bounded 0–1: `Cp`, `pct_lactating`, fractions |
| **triangular** | `mc2d::rtriang(n, min=lower, mode=mean, max=upper)` | Expert-elicited; only min/mode/max known; `Ca`, `C_growth`, `DE_pct` |
| **pert** | `mc2d::rpert(n, min=lower, mode=mean, max=upper, shape=4)` — internally Beta with alpha = 1 + 4x(mode-min)/(max-min), beta = 1 + 4x(max-mode)/(max-min) | IPCC coefficients where mode is well-established but tails uncertain: `Cfi`, `Ym`, `Bo`, `EF3_PRP`, `Frac_GASM` |
| **uniform** | `runif(n, lower, upper)` | When only range is known; maximum uncertainty |
| **constant** | `rep(mean, n)` | Zero uncertainty; e.g. `weight_gain = 0` for mature cows; used in decomposition runs |
| **tnorm\_0\_1** | Normal, then clamped to [0,1] | Fractions that must stay in [0,1] |

## 6. Monte Carlo Sampling Engine — Gaussian Copula

**R:** `generate_mc_samples()` in `R/mc_sampling.R`

The challenge: activity data parameters must be **(a) correlated with each other** and **(b) each follow its own distribution type**. The Gaussian copula solves this. The same approach is optionally applied to emission factors (§6.5).

### Step 1 — Split parameters
```r
ad_params <- param_specs[ param_specs$param_type == "activity_data", ]
ef_params <- param_specs[ param_specs$param_type == "emission_factor", ]
```

### Step 2 — Activity data: correlated sampling (if correlation matrix provided)

A private helper `.copula_sample(n_iter, params, corr_mat)` is called when `corr_matrix` is non-null and dimensionally consistent with the AD parameter block.

**2a. Fix the correlation matrix:**
```r
corr_matrix <- Matrix::nearPD(corr_matrix, corr = TRUE)$mat
```
`nearPD` finds the nearest positive semi-definite matrix — required for multivariate normal sampling.

**2b. Draw correlated standard-normal samples:**
```r
Z <- MASS::mvrnorm(n_iter, mu = rep(0, n_ad), Sigma = corr_matrix)
# Z is n_iter x n_ad; columns are correlated per corr_matrix
```

**2c. Transform to uniform [0, 1] via the normal CDF:**
```r
U <- pnorm(Z)
# Each u[i,j] in (0,1). The correlation structure is preserved.
```

**2d. Transform each column to its target distribution via the inverse CDF:**
```r
for (j in 1:n_ad) {
  ad_samples[, j] <- transform_marginal(U[, j],
                       distribution = ad_params$distribution[j],
                       mean_val = ad_params$mean[j],
                       lower = ad_params$lower[j],
                       upper = ad_params$upper[j])
}
```

**Key insight:** Normal → Uniform → Target preserves the correlation rank-structure from step 2b while giving each parameter any desired marginal distribution. This is the standard Gaussian copula / Iman-Conover method.

**If no correlation matrix:** each AD parameter is sampled independently via `.indep_sample()`.

### Step 3 — Emission factors: independent or uniform equicorrelation (see §6.5)
```r
ef_samples <- if (use_ef_corr) {
  .copula_sample(n_iter, ef_params, ef_corr_matrix)
} else {
  .indep_sample(n_iter, ef_params)
}
```

**Output:** `samples` data frame, `n_iter x (n_ad + n_ef)` columns, one column per parameter.

### 6.5 Emission Factor Correlations — Uniform Equicorrelation

**R:** `make_uniform_corr(n, rho)` in `R/mc_sampling.R`

By default, emission factors are sampled independently (IPCC standard Approach 2 assumption). However, if the IPCC methodology itself introduces a systematic bias — for example, if the `Ym%` equation tends to underestimate CH4 across all cattle types simultaneously — all EFs would be over- or under-estimated together. This can be captured by a **uniform equicorrelation matrix**:

```
R_EF = (1 - rho) x I  +  rho x 11^T
```

where `I` is the n_ef x n_ef identity matrix and `11^T` is the all-ones matrix. Every pair of EFs has the same off-diagonal correlation `rho`. The matrix is passed through `Matrix::nearPD()` to ensure positive semi-definiteness.

**Construction in the server:**
```r
ef_corr_reactive <- reactive({
  req(rv$param_specs)
  if (input$ef_corr_mode == "none") return(NULL)
  ef_params <- rv$param_specs[rv$param_specs$param_type == "emission_factor", ]
  n_ef <- nrow(ef_params)
  if (n_ef < 2) return(NULL)
  rho <- input$ef_corr_rho         # slider: 0.0 to 0.9, default 0.3
  mat <- make_uniform_corr(n_ef, rho)
  rownames(mat) <- colnames(mat) <- ef_params$parameter
  mat
})
```

**User interface (Tab 4 — Emission Factor Correlations):**
- Mode: "No EF correlations (default)" | "Uniform EF correlation"
- Slider: rho from 0.0 to 0.9 (default 0.3)
- Heatmap preview of the resulting matrix

**Interpretation:**
- rho = 0: fully independent (identical to no-correlation run; IPCC default)
- rho = 0.3: moderate systematic bias assumption (all EFs shift together with 30 % correlation)
- rho > 0.5: strong assumption; requires explicit justification

The `ef_corr_matrix` is passed through the same Gaussian copula pipeline as the AD correlation matrix. Its effect is to widen the combined EF-only uncertainty distribution compared with the independent case.

## 7.5 QA/QC Module

**R:** `run_qaqc(param_specs)` in `R/utils_qaqc.R`

After data upload, Tab 2 runs 6 automated checks on every parameter row. Results are shown as a traffic-light table (pass/warn/fail).

| Check | Key | Logic | Severity |
|---|---|---|---|
| **Bounds order** | `bounds_order` | Requires `lower < mean < upper`. Skipped for `constant` distribution. | fail |
| **Non-negative lower** | `non_negative` | `lower >= 0` (all IPCC livestock parameters must be non-negative). | warn |
| **Range constraints** | `range_check` | DE\_pct in [1, 100] — fail if outside; Ym\_pct in [1, 15] — warn if outside; fraction parameters in [0, 1] — fail if outside. | fail / warn |
| **Distribution suitability** | `dist_suitability` | Beta: mean must be in (0,1) and bounds in [0,1]. Lognormal: mean must be strictly positive. tnorm\_0\_1: warns if bounds extend beyond [0,1]. | fail / warn |
| **Benchmark deviation** | `benchmark_deviation` | Compares mean against `ipcc_default` from `PARAM_CATALOGUE`. >50 % deviation = warn; >200 % deviation = fail. `cattle_pop` is excluded (no IPCC default). | fail / warn |
| **Asymmetric bounds** | `asymmetric_bounds` | For EF3\_PRP, EF4, EF5, Frac\_LEACH: checks that `(upper - mean) / (mean - lower) >= 1.5`. Near-symmetric bounds underestimate the upper tail per Penman et al. (2000) / Monni et al. (2007). | warn |

The check runs are stored in a tidy data frame (one row per group × parameter × check), so results can be filtered by status or parameter. The summary badges show total counts of pass / warn / fail.

**Action required:** Fix all `fail` results before running the simulation. `warn` results are advisory — document your justification (e.g. country-specific value outside IPCC typical range).

## 8. Multi-System Inventory

**R:** `run_inventory_simulation()` in `R/mc_simulation.R`

For inventories with multiple production systems (e.g. Pastoral + Semi-intensive + Intensive):

1. Each system (identified by the combination of `cattle_type`, `aggregation_level`, `sub_category`) is simulated independently: its own parameter values, its own correlation matrix (expanded from `rv$corr_matrix` to match that system's AD parameters), its own seed = `base_seed + system_index`.
2. For each iteration `i`, results are **summed across all systems**:

```r
inventory_results$total_co2e[i] = system_1$results$total_co2e[i]
                                 + system_2$results$total_co2e[i]
                                 + ... + system_k$results$total_co2e[i]
```

3. Uncertainty metrics are computed on the summed distribution.

This correctly propagates uncertainty from all sub-categories into the national total. Systems simulated independently means within-system correlations are captured but cross-system correlations (e.g. dairy cows and beef heifers moving together) are not — they are treated as independent.

**Note:** Uncertainty decomposition (§10) runs only when there is exactly one system group. It is skipped for multi-system inventories.

## 9. Uncertainty Metrics

**R:** `calc_uncertainty_metrics(x)` in `R/mc_uncertainty.R`

After N iterations we have a vector `x` of N values for each output (e.g. `total_co2e`). The following are computed:

| Metric | Formula | IPCC use |
|---|---|---|
| **mean** | Sum(x_i) / N | Best estimate to report |
| **median** | 50th percentile | Robust central tendency |
| **sd** | sqrt[Sum(x_i - mean)^2 / (N-1)] | Absolute spread |
| **cv\_pct** | sd / mean x 100 | **Primary IPCC metric** — relative uncertainty %; compare across categories and countries |
| **ci\_lower** | quantile(x, 0.025) | Lower bound of 95 % CI |
| **ci\_upper** | quantile(x, 0.975) | Upper bound of 95 % CI |
| **moe\_pct** | 1.96 x (sd / sqrt(N)) / mean x 100 | Precision of the MC mean itself (shrinks with more iterations) |
| **iqr** | Q75 - Q25 | Middle 50 % spread |
| **min / max** | Observed extremes | Extreme tails |

The 95 % CI is reported as `[ci_lower, ci_upper]` and the CV % as the headline uncertainty figure. In IPCC reporting format: "Enteric fermentation = 15 200 t CO2eq, CV = 13 %, 95 % CI: 12 100–18 800".

## 10. Uncertainty Decomposition: AD vs EF

**R:** `decompose_uncertainty()` in `R/mc_uncertainty.R`

Answers: *how much of the total uncertainty comes from not knowing the animal numbers/characteristics (AD) vs. not knowing the IPCC conversion coefficients (EF)?*

Three separate Monte Carlo runs are executed (using the same seed for reproducibility):

### Run 1 — Combined (full uncertainty)
All parameters varied according to their distributions. Gives the true total uncertainty.

### Run 2 — AD-only
Emission factor rows are set to `constant` (zero variance). Only activity data varies.

### Run 3 — EF-only
Activity data held at their means. Only emission factors vary. `corr_matrix = NULL` (no point correlating fixed values).

### Reading the results

| Run | CV % | What it means |
|---|---|---|
| AD-only | e.g. 10 % | Not knowing animal numbers/characteristics alone causes 10 % relative uncertainty |
| EF-only | e.g. 8 % | Not knowing IPCC coefficients alone causes 8 % uncertainty |
| Combined | e.g. 13 % | Together they cause 13 % — **not simply sqrt(10^2 + 8^2)** because equations are non-linear |

This maps to IPCC Table 3.3:

| Source | IPCC Code | Gas | Tier | AD uncert. | EF uncert. | Combined |
|---|---|---|---|---|---|---|
| Enteric fermentation | 3.A.1 | CH4 | 2 | CV from AD-only | CV from EF-only | CV from combined |
| Manure management | 3.A.2 | CH4+N2O | 2 | CV from AD-only | CV from EF-only | CV from combined |

The decomposition bar chart in Tab 6 visualises these three CV % values side by side.

## 11. Sensitivity Analysis: Which Parameters Drive Uncertainty?

**R:** `sensitivity_analysis()` in `R/mc_sensitivity.R`

After the Monte Carlo, we have an N x p matrix of sampled input values and N output values. Two methods are provided. The sensitivity tab includes a toggle (`uiOutput("sens_view_toggle")`) to switch between the main run (with correlations) and the comparison run (without correlations), when both are available.

### 11.1 Standardised Regression Coefficients (SRC)
**R:** `calc_src(inputs, output)`

```r
inputs_std <- scale(inputs)           # z-scores: (x - mean) / sd
output_std <- scale(output)
fit        <- lm(output_std ~ ., data = cbind(inputs_std, y = output_std))
SRC[j]     <- coef(fit)[j]            # coefficient for parameter j
```

**Interpretation:** SRC = +0.65 for `cattle_pop` means: a 1-SD increase in population → 0.65-SD increase in total emissions. Parameters are ranked by |SRC|. The R2 of the regression tells you what fraction of output variance is explained by a linear model — if R2 < 0.7, the model is substantially non-linear and SRC is less reliable.

### 11.2 Partial Rank Correlation Coefficients (PRCC)
**R:** `calc_prcc(inputs, output)`

```r
inputs_ranked  <- lapply(inputs, rank)     # replace values with ranks
output_ranked  <- rank(output)

for (j in 1:p) {
  # Regress parameter j ranks on all OTHER parameters' ranks → residuals e_j
  # Regress output ranks on all OTHER parameters' ranks → residuals e_y
  PRCC[j] <- cor(e_j, e_y)               # Pearson on the residuals
}
```

**Interpretation:** PRCC measures the monotonic relationship between parameter j and the output **after removing the effect of all other parameters**. It is robust to non-linearity as long as the relationship is monotonic. |PRCC| → 1 = highly influential; |PRCC| → 0 = negligible.

Only parameters with non-zero variance are included. Constant parameters (decomposition runs) are automatically excluded.
